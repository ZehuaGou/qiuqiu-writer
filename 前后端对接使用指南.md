## 📖 使用步骤

### 1. 打开拆书分析页面

在浏览器中访问：
```
http://localhost:5173
```

然后导航到**拆书分析工具**页面。

### 2. 检查AI服务状态

页面顶部会显示AI服务连接状态：
- ✅ **AI已连接** (绿色) - 服务正常，可以开始分析
- ⚠️ **检查中...** (黄色) - 正在检查连接
- ❌ **AI未连接** (红色) - 服务未启动，需要先启动后端

### 3. 上传小说文件

1. 点击 **"选择 TXT 文件"** 按钮
2. 选择一个 `.txt` 格式的小说文件
3. 文件名会显示在上传区域

### 4. 设置章节拆分

1. 选择每组章节数（10/20/50/100/200章）
2. 点击 **"执行拆分"** 按钮
3. 等待拆分完成，会显示所有章节组

### 5. 选择要分析的章节

1. 在章节组列表中选择要分析的章节（默认选中前3组）
2. 可以使用 **"全选"** 或 **"取消全选"** 按钮
3. 查看每组的字符数

### 6. 开始AI分析

1. 点击 **"开始 AI 分析"** 按钮
2. 观察分析进度：
   - 左侧显示当前正在分析的章节组
   - 进度条显示整体完成进度
   - 右侧**实时显示**AI生成的分析内容（流式输出）

### 7. 查看和导出结果

1. 分析完成后，所有结果显示在右侧面板
2. 每个章节组的分析包括：
   - 章节号
   - 章节标题
   - 核心剧情梗概
   - 画面感/镜头序列
   - 关键情节点
   - 章节氛围/情绪
   - 结尾钩子
3. 点击右上角的 **下载图标** 导出所有分析结果为Markdown文件

## 🔧 技术实现

### 前端集成

在 `BookSplitterPage.tsx` 中：

```typescript
// 1. 导入API客户端
import { analyzeChapterContent, testAPIConnection } from '../utils/bookAnalysisApi';

// 2. 检查API连接
useEffect(() => {
  const checkAPIConnection = async () => {
    const result = await testAPIConnection();
    setApiStatus(result.success ? 'connected' : 'error');
  };
  checkAPIConnection();
}, []);

// 3. 调用分析接口
const analysisResult = await analyzeChapterContent(
  group.content,
  (progress) => {
    // 实时更新UI
    if (progress.text) {
      setAnalysisResults(prev => ({
        ...prev,
        [group.name]: {
          ...prev[group.name],
          content: prev[group.name].content + progress.text
        }
      }));
    }
  },
  {
    model: 'codedrive-chat',
    temperature: 0.7,
    maxTokens: 4000
  }
);
```

## ⚠️ 注意事项

1. **API限流**: 
   - 连续分析多个章节组时，系统会自动添加1秒延迟
   - 避免触发API限流

2. **内容长度**: 
   - 建议每组不超过50章
   - 单次请求token限制为4000

3. **网络连接**: 
   - 确保能访问 `http://10.96.20.92/v1`
   - 检查防火墙设置

4. **服务状态**: 
   - 分析前检查顶部的AI连接状态
   - 红色表示需要启动后端服务

## 🐛 故障排除

### 问题1: AI未连接

**症状**: 页面显示"AI未连接"

**解决方案**:
```bash
# 检查后端服务是否运行
lsof -i :8001

# 如果没有运行，启动服务
cd /home/pang/writerAI/backend
python3 -m memos.api.ai_api --port 8001
```

### 问题2: 分析请求失败

**症状**: 显示"分析失败"错误

**解决方案**:
1. 检查后端日志: `tail -f /tmp/ai_api.log`
2. 验证API密钥和端点配置
3. 测试健康检查接口: `curl http://localhost:8001/ai/health`

### 问题3: 流式响应中断

**症状**: 分析过程中突然停止

**解决方案**:
1. 检查网络连接
2. 查看浏览器控制台错误
3. 减少每组的章节数
