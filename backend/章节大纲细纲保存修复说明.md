# 章节大纲和细纲保存修复说明

## 问题描述

章节中的大纲和细纲没有保存到后端的传递和保存存在问题。

## 修复内容

### 1. 更新了 `ChapterResponse` 模型

添加了 `metadata` 字段，确保大纲和细纲能正确返回：

```python
class ChapterResponse(BaseModel):
    # ... 其他字段
    metadata: Optional[Dict[str, Any]] = {}  # 包含outline和detailed_outline
    summary: Optional[str] = None
    # ...
```

### 2. 更新了 `ChapterUpdate` 模型

支持通过API更新大纲和细纲：

```python
class ChapterUpdate(BaseModel):
    # ... 其他字段
    chapter_metadata: Optional[Dict[str, Any]] = None  # 包含outline和detailed_outline
    # ...
```

### 3. 增强了Prompt模板

#### 创建了新的章节分析Prompt模板（JSON格式）

- **文件**: `backend/scripts/init_chapter_analysis_prompt.py`
- **类型**: `chapter_analysis`
- **格式**: JSON（包含大纲和细纲）
- **特点**: 返回结构化数据，而不是Markdown表格

#### Prompt模板要求

新的prompt模板明确要求AI返回包含以下字段的JSON：

```json
{
  "chapter_number": 1,
  "title": "章节标题",
  "summary": "章节概要",
  "outline": {
    "core_function": "核心功能",
    "key_points": ["关键点1", "关键点2"],
    "visual_scenes": ["画面1", "画面2"],
    "atmosphere": ["氛围1", "氛围2"],
    "hook": "结尾钩子"
  },
  "detailed_outline": {
    "sections": [
      {
        "section_number": 1,
        "title": "小节标题",
        "content": "小节内容"
      }
    ]
  }
}
```

### 4. 添加了JSON解析功能

#### 新增方法：`parse_single_chapter_response()`

用于解析单个章节的JSON响应：

```python
def parse_single_chapter_response(self, ai_response: str) -> Dict[str, Any]:
    """解析单个章节的AI响应，提取JSON数据"""
    # 支持JSON代码块和纯JSON格式
    # 如果解析失败，返回None（可能是Markdown格式）
```

### 5. 更新了 `analyze_chapter` 接口

在流式响应完成后，自动尝试解析JSON数据：

- 如果prompt返回JSON格式，会解析并发送 `structured_data` 消息
- 包含完整的大纲和细纲数据
- 前端可以接收并使用这些数据

### 6. 添加了调试日志

在保存章节时，会记录：
- 是否有大纲数据
- 是否有细纲数据
- 保存成功的信息

## 使用方法

### 1. 初始化新的Prompt模板

运行初始化脚本：

```bash
cd backend
python scripts/init_chapter_analysis_prompt.py
```

这会创建一个新的 `chapter_analysis` 类型的默认模板，返回JSON格式。

### 2. 前端调用

#### 方式1：使用 `analyzeChapterContent`（单章分析）

```typescript
const result = await analyzeChapterContent(
  content,
  (progress) => {
    // 接收流式响应
    if (progress.text) {
      
    }
    
    // 如果收到结构化数据
    // 注意：需要在SSE消息中监听 structured_data 类型
  }
);
```

#### 方式2：使用 `analyzeBookEnhanced`（增强分析，自动创建作品）

```typescript
await analyzeBookEnhanced(
  content,
  true, // autoCreateWork
  (progress) => {
    if (progress.workResult) {
      
    }
  }
);
```

#### 方式3：使用 `analyzeChaptersIncremental`（渐进式分析）

```typescript
await analyzeChaptersIncremental(
  content,

  }

```

### 3. 前端接收结构化数据

在SSE消息处理中，监听 `structured_data` 类型：

```typescript

  // 可以使用这些数据更新UI或保存到本地

```

## 验证步骤

1. **检查Prompt模板**
   ```bash
   curl http://localhost:8001/api/v1/prompt-templates/type/chapter_analysis/default
   ```
   确认返回的prompt要求JSON格式。

2. **调用分析接口**
   使用包含章节内容的文本调用 `/ai/analyze-chapter`。

3. **检查日志**
   查看后端日志，确认：
   - "章节包含完整的大纲和细纲数据"
   - "✅ 章节 X 已保存，大纲: True, 细纲: True"

4. **查询章节数据**
   ```bash
   curl http://localhost:8001/api/v1/chapters/{chapter_id}
   ```
   检查返回的 `metadata` 字段是否包含 `outline` 和 `detailed_outline`。

## 注意事项

1. **Prompt模板版本**
   - 旧的Markdown表格格式仍然支持（向后兼容）
   - 新的JSON格式需要运行初始化脚本创建模板

2. **数据格式**
   - 如果AI返回Markdown格式，不会解析结构化数据
   - 只有JSON格式才会解析并保存大纲和细纲

3. **向后兼容**
   - 如果数据库中没有新的prompt模板，会使用代码中的默认prompt
   - 默认prompt返回Markdown格式，不会包含结构化的大纲和细纲
