# 路由404最终解决方案

## 问题确认

`/api/v1/works` 返回 404 是因为路由注册失败。根本原因是：
- 导入路由模块时触发 `memos.__init__.py`
- `memos.__init__.py` 需要 `pydantic`
- `pydantic` 可能未安装在当前Python环境

## 解决方案

### ✅ 方案1：使用虚拟环境（推荐）

项目有 `.venv` 虚拟环境，确保使用它：

```bash
cd backend

# 激活虚拟环境
source .venv/bin/activate

# 安装依赖（如果还没安装）
pip install pydantic fastapi sqlalchemy asyncpg python-jose passlib

# 启动服务
python -m uvicorn memos.api.ai_api:app --host 0.0.0.0 --port 8001
```

### ✅ 方案2：使用启动脚本（最简单）

启动脚本会自动检测并使用虚拟环境：

```bash
cd backend
./start_ai_api.sh
```

脚本会：
1. 检查是否有 `.venv`
2. 如果有，自动激活
3. 使用虚拟环境中的Python运行服务

### ✅ 方案3：使用Poetry

如果使用Poetry管理依赖：

```bash
cd backend
poetry install  # 安装所有依赖
poetry run uvicorn memos.api.ai_api:app --host 0.0.0.0 --port 8001
```

## 验证步骤

### 1. 检查虚拟环境

```bash
cd backend
source .venv/bin/activate
which python  # 应该指向 .venv/bin/python
python -c "import pydantic; print('✅ pydantic available')"
```

### 2. 测试路由导入

```bash
source .venv/bin/activate
python -c "
import sys
sys.path.insert(0, 'src')
import importlib
module = importlib.import_module('memos.api.routers.works_router')
print('✅ Works router imported')
print(f'Router prefix: {module.router.prefix}')
"
```

### 3. 启动服务并查看日志

```bash
./start_ai_api.sh
```

查看启动日志，应该看到：
```
📋 Works router prefix: /api/v1/works
📋 Works router routes count: X
✅ WriterAI application routers registered successfully
```

### 4. 验证路由

访问 `http://localhost:8001/docs`，应该能看到 `/api/v1/works` 相关的端点。

## 如果仍然404

### 检查启动日志

查看服务启动时的完整日志，查找：
- `❌ WriterAI application routers not available` - 如果有，查看后面的错误信息
- `ModuleNotFoundError: No module named 'pydantic'` - 说明依赖未安装

### 手动测试

```bash
# 激活虚拟环境
source .venv/bin/activate

# 测试导入
python -c "
import sys
sys.path.insert(0, 'src')
try:
    from memos.api.routers.works_router import router
    print('✅ Router imported successfully')
    print(f'Prefix: {router.prefix}')
    print(f'Routes: {len(router.routes)}')
except Exception as e:
    print(f'❌ Error: {e}')
    import traceback
    traceback.print_exc()
"
```

## 关键点

1. **必须使用虚拟环境**：确保所有依赖都在虚拟环境中
2. **检查启动脚本**：`start_ai_api.sh` 应该自动激活虚拟环境
3. **查看日志**：启动日志会显示路由是否注册成功
4. **安装依赖**：如果虚拟环境中缺少依赖，需要安装

## 快速修复命令

```bash
cd backend

# 激活虚拟环境
source .venv/bin/activate

# 安装缺失的依赖
pip install pydantic fastapi sqlalchemy asyncpg python-jose passlib

# 重启服务
./start_ai_api.sh
```

## 相关文件

- `backend/start_ai_api.sh` - 启动脚本（会自动检测虚拟环境）
- `backend/src/memos/api/ai_api.py` - 路由注册代码
- `backend/src/memos/__init__.py` - 包初始化（需要pydantic）
