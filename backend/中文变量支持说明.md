# Prompt模板中文变量支持说明

## 概述

系统现在支持在Prompt模板中使用中文变量名，并且可以从作品和章节的metadata中根据键获取数据。

## 支持的变量类型

### 1. 中文变量名

所有英文变量都有对应的中文变量名，方便前端用户编辑时使用：

| 英文变量 | 中文变量 | 说明 |
|---------|---------|------|
| `{work_title}` | `{作品标题}` | 作品标题 |
| `{work_description}` | `{作品描述}` | 作品描述 |
| `{work_genre}` | `{作品类型}` | 作品类型 |
| `{all_characters}` | `{所有角色}` | 所有角色列表（格式化字符串） |
| `{chapter_characters}` | `{章节角色}` | 当前章节使用的角色列表 |
| `{current_chapter_title}` | `{章节标题}` | 当前章节标题 |
| `{current_chapter_number}` | `{章节号}` | 当前章节号 |
| `{current_chapter_summary}` | `{章节摘要}` | 当前章节摘要 |
| `{content}` | `{章节内容}` | 当前章节内容 |
| `{previous_chapters_summary}` | `{前文摘要}` | 前文章节摘要 |
| `{previous_chapters_content}` | `{前文内容}` | 前文内容 |
| `{previous_outlines}` | `{前文大纲}` | 前文大纲 |
| `{previous_detailed_outlines}` | `{前文细纲}` | 前文细纲 |
| `{locations}` | `{地点}` | 地点列表 |
| `{outline}` | `{大纲}` | 当前章节的大纲（JSON格式） |

### 2. Metadata访问

支持从作品和章节的metadata中根据键获取数据：

- **作品metadata**: `{作品.键名}`
- **章节metadata**: `{章节.键名}`

#### 使用示例

假设作品的`work_metadata`中有：
```json
{
  "世界观": "现代都市",
  "时代背景": "2024年",
  "主要冲突": "科技与传统的碰撞"
}
```

章节的`chapter_metadata`中有：
```json
{
  "核心冲突": "主角面临选择",
  "情感基调": "紧张",
  "场景设置": "办公室"
}
```

在Prompt模板中可以这样使用：

```markdown
# 章节生成

作品背景：{作品.世界观}
时代：{作品.时代背景}
主要冲突：{作品.主要冲突}

当前章节信息：
- 核心冲突：{章节.核心冲突}
- 情感基调：{章节.情感基调}
- 场景设置：{章节.场景设置}

已有角色：
{所有角色}

请根据以上信息生成章节内容。
```

## 实现原理

### 1. PromptTemplate.format_prompt方法

修改了`PromptTemplate`类的`format_prompt`方法，使用正则表达式匹配所有变量（包括中文），并支持`{对象.键}`格式的metadata访问。

```python
def format_prompt(self, **kwargs) -> str:
    """格式化提示词，替换变量
    支持英文变量名：{variable_name}
    支持中文变量名：{所有角色}、{章节角色}等
    支持metadata访问：{作品.xxx}、{章节.xxx}
    """
    # 使用正则表达式匹配 {变量名} 或 {对象.键} 格式
    pattern = r'\{([^}]+)\}'
    # ... 替换逻辑
```

### 2. PromptContextService.format_prompt方法

在`format_prompt`方法中，同时提供英文和中文变量名，并将作品和章节的metadata作为对象传递：

```python
# 中文变量
vars_dict["所有角色"] = all_chars_formatted
vars_dict["章节角色"] = chapter_chars_formatted

# Metadata对象
vars_dict["作品"] = work_metadata  # 支持 {作品.xxx}
vars_dict["章节"] = chapter_metadata  # 支持 {章节.xxx}
```

## 使用示例

### 示例1：角色生成Prompt

```markdown
# 角色生成

根据以下信息生成新角色：

作品信息：
- 标题：{作品标题}
- 类型：{作品类型}
- 世界观：{作品.世界观}
- 时代背景：{作品.时代背景}

已有角色：
{所有角色}

前文摘要：
{前文摘要}

请生成一个新角色，要求：
1. 与已有角色有所区别
2. 符合作品的世界观和时代背景
3. 与前文内容连贯
```

### 示例2：章节生成Prompt

```markdown
# 章节生成

根据以下信息生成新章节：

作品：{作品标题}
当前章节：第{章节号}章 - {章节标题}

已有角色：
{所有角色}

当前章节使用的角色：
{章节角色}

前文内容：
{前文内容}

前文大纲：
{前文大纲}

章节设置：
- 核心冲突：{章节.核心冲突}
- 情感基调：{章节.情感基调}
- 场景设置：{章节.场景设置}

请生成下一章的内容，要求与前文连贯。
```

### 示例3：章节总结Prompt

```markdown
# 章节总结

请总结以下章节：

章节标题：{章节标题}
章节号：第{章节号}章

章节内容：
{章节内容}

请用2-3句话概括本章的主要内容。
```

## 前端编辑建议

1. **使用中文变量名**：推荐使用中文变量名，更直观易懂
2. **利用metadata**：可以在作品和章节的metadata中存储自定义信息，然后在prompt中通过`{作品.xxx}`和`{章节.xxx}`访问
3. **变量提示**：前端可以提供变量提示列表，帮助用户选择合适的变量

## 注意事项

1. **变量不存在**：如果变量不存在或值为空，会被替换为空字符串
2. **metadata键名**：使用`{作品.xxx}`和`{章节.xxx}`时，确保metadata中存在对应的键
3. **大小写敏感**：变量名和metadata键名都是大小写敏感的
4. **嵌套访问**：目前不支持嵌套访问（如`{作品.角色.主角}`），如果需要，可以在metadata中存储JSON字符串

## 扩展建议

如果需要支持更复杂的metadata访问（如嵌套对象、数组等），可以扩展`format_prompt`方法，支持类似`{作品.角色.主角.姓名}`这样的路径访问。
