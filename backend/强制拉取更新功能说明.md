# 强制拉取更新功能说明

## 功能概述

添加了一个"拉取更新"按钮，允许用户手动从服务器强制拉取最新内容，忽略本地缓存。

## 实现内容

### 1. 后端方法

在 `sharedbClient.ts` 中添加了 `forcePullFromServer()` 方法：

```typescript
async forcePullFromServer(documentId: string): Promise<ShareDBDocument | null> {
  // 直接从服务器获取，忽略本地缓存
  const serverDoc = await this.fetchFromServer(documentId);
  
  // 更新本地状态和缓存
  // 通知编辑器更新内容
  return serverDoc;
}
```

### 2. 前端按钮

在 `NovelEditorPage.tsx` 中：

1. **添加了图标导入**：
   ```typescript
   import { RefreshCw } from 'lucide-react';
   ```

2. **添加了处理函数**：
   ```typescript
   const handleForcePull = async () => {
     // 从服务器强制拉取最新内容
     const serverDoc = await sharedbClient.forcePullFromServer(documentId);
     // 更新编辑器内容
     editor.commands.setContent(serverContent);

   ```

3. **添加了工具栏按钮**：
   ```tsx
   <button
     className="toolbar-btn"
     onClick={handleForcePull}
     title="强制从服务器拉取最新内容"
   >
     <RefreshCw size={16} />
     <span>拉取更新</span>
   </button>
   ```

## 功能特点

### 1. 忽略本地缓存
- 直接从服务器获取最新内容
- 不依赖本地缓存
- 确保获取的是服务器上的最新版本

### 2. 自动更新编辑器
- 拉取成功后自动更新编辑器内容
- 通过回调机制通知编辑器
- 更新本地状态和缓存

### 3. 错误处理
- 如果服务器没有文档，返回 null
- 如果网络错误，抛出异常
- 记录详细的日志

## 使用场景

### 场景1：看不到其他用户的更改
- 用户A编辑内容，保存到服务器
- 用户B打开文档，但看不到A的更改
- **解决方案**：点击"拉取更新"按钮，强制从服务器获取最新内容

### 场景2：本地缓存过期
- 本地缓存可能是旧版本
- 自动同步可能失败
- **解决方案**：手动点击"拉取更新"，强制刷新

### 场景3：网络问题恢复后
- 网络断开后恢复
- 需要获取服务器上的最新内容
- **解决方案**：点击"拉取更新"，确保内容是最新的

## 按钮位置

按钮位于编辑器工具栏中，在格式按钮（H1、粗体、下划线）之后，使用分隔符分隔。

## 注意事项

1. **会覆盖本地未保存的更改**：
   - 如果本地有未保存的更改，拉取更新会覆盖这些更改
   - 建议在拉取前先保存本地更改

2. **网络延迟**：
   - 从服务器拉取需要时间
   - 按钮点击后可能需要等待几秒

3. **错误处理**：
   - 如果服务器没有文档，会显示警告
   - 如果网络错误，会抛出异常
