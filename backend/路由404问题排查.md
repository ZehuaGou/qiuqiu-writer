# 路由404问题排查

## 问题描述

访问 `/api/v1/works?page=1&size=20` 返回 404 错误。

## 可能原因

### 1. 路由注册失败

路由注册在 `try-except` 块中，如果导入或注册时出错，会被捕获并记录warning，但不会抛出异常。

**检查方法**：
查看启动日志，看是否有：
```
⚠️  WriterAI application routers not available: ...
```

### 2. 认证问题

`/api/v1/works` 路由需要认证（`get_current_user_id`），如果：
- 没有提供token
- token无效
- token过期

可能会返回401或403，但如果是路由未注册，会返回404。

### 3. 路径不匹配

检查：
- 路由前缀：`/api/v1/works`
- 实际请求路径：`/api/v1/works?page=1&size=20`

## 排查步骤

### 1. 检查路由是否注册

在启动日志中查找：
```
✅ WriterAI application routers registered successfully
```

如果没有这条日志，说明路由注册失败。

### 2. 检查导入错误

路由注册代码：
```python
try:
    from memos.api.routers import (
        get_auth_router,
        get_chapters_router,
        get_templates_router,
        get_works_router,
    )
    
    app.include_router(get_auth_router())
    app.include_router(get_chapters_router())
    app.include_router(get_templates_router())
    app.include_router(get_works_router())
    logger.info("✅ WriterAI application routers registered successfully")
except Exception as e:
    logger.warning(f"⚠️  WriterAI application routers not available: {e}", exc_info=True)
```

如果出现异常，会记录warning。

### 3. 检查路由定义

路由定义在 `backend/src/memos/api/routers/works_router.py`:
```python
router = APIRouter(prefix="/api/v1/works", tags=["作品管理"])

@router.get("/", response_model=WorkListResponse)
async def list_works(...):
    ...
```

### 4. 测试路由

使用curl测试：
```bash
# 测试不带认证（应该返回401或404）
curl http://localhost:8001/api/v1/works?page=1&size=20

# 测试带认证
curl -H "Authorization: Bearer YOUR_TOKEN" http://localhost:8001/api/v1/works?page=1&size=20
```

## 解决方案

### 方案1：检查启动日志

查看完整的启动日志，确认路由是否注册成功。

### 方案2：添加调试日志

在 `ai_api.py` 中添加更详细的日志：

```python
# 注册WriterAI应用路由
try:
    from memos.api.routers import (
        get_auth_router,
        get_chapters_router,
        get_templates_router,
        get_works_router,
    )
    
    auth_router = get_auth_router()
    chapters_router = get_chapters_router()
    templates_router = get_templates_router()
    works_router = get_works_router()
    
    logger.info(f"Works router prefix: {works_router.prefix}")
    logger.info(f"Works router routes: {[r.path for r in works_router.routes]}")
    
    app.include_router(auth_router)
    app.include_router(chapters_router)
    app.include_router(templates_router)
    app.include_router(works_router)
    logger.info("✅ WriterAI application routers registered successfully")
except Exception as e:
    logger.error(f"❌ WriterAI application routers not available: {e}", exc_info=True)
```

### 方案3：检查FastAPI路由列表

在应用启动后，打印所有注册的路由：

```python
@app.on_event("startup")
async def startup_event():
    logger.info("=== 注册的路由 ===")
    for route in app.routes:
        if hasattr(route, 'path') and hasattr(route, 'methods'):
            logger.info(f"{list(route.methods)} {route.path}")
```

## 常见问题

### 1. 导入错误

如果 `works_router.py` 中有导入错误，路由注册会失败。

**检查**：
- 所有依赖是否已安装
- 导入路径是否正确
- 是否有循环导入

### 2. 认证依赖

`get_current_user_id` 如果抛出异常，可能导致路由无法访问。

**检查**：
- 认证中间件是否正常工作
- token验证逻辑是否正确

### 3. 路由冲突

如果有其他路由使用了相同的路径，可能导致冲突。

**检查**：
- 是否有其他路由使用 `/api/v1/works`
- 路由顺序是否正确

## 快速修复

如果路由注册失败，可以：

1. **检查异常信息**：查看启动日志中的异常堆栈
2. **修复导入错误**：确保所有依赖正确导入
3. **重启服务**：修复后重启服务

## 验证

修复后，访问：
```
http://localhost:8001/docs
```

在Swagger UI中应该能看到 `/api/v1/works` 相关的端点。
