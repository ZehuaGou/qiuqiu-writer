# 本地缓存覆盖问题修复

## 问题描述

**严重的数据丢失问题**：本地缓存导致数据回写，覆盖了其他用户的更改。

### 问题场景

1. 用户A编辑第一段，保存到本地缓存
2. 用户A关闭浏览器
3. 用户B编辑第二段，保存到服务器
4. 用户A重新打开浏览器
5. **问题**：用户A的本地缓存（只有第一段）覆盖了服务器上的内容（第一段+第二段），导致第二段丢失

## 根本原因

1. **`getDocument()` 优先使用本地缓存**：从缓存返回旧数据，没有检查服务器版本
2. **同步时使用旧baseContent**：即使检测到服务器版本更新，仍使用本地旧内容作为baseContent
3. **版本号不准确**：使用缓存的旧版本号，导致后端认为没有冲突

## 修复方案

### 1. 修改 `getDocument()` - 优先从服务器获取

**修复前**：
- 先从本地缓存获取
- 如果在线，异步检查更新
- 缓存可能包含旧数据

**修复后**：
- **总是先尝试从服务器获取最新版本**
- 只有服务器获取失败时，才使用本地缓存
- 使用缓存时，将版本号减1，强制在同步时检查服务器

```typescript
// 关键修复：总是先尝试从服务器获取最新版本
let serverDoc = await this.fetchFromServer(documentId);
if (serverDoc) {
  // 使用服务器最新数据
  return serverDoc;
}

// 只有服务器获取失败时，才使用本地缓存
// 但标记版本为更小的值，强制在同步时检查服务器
this.currentVersion.set(documentId, Math.max(0, cachedVersion - 1));
```

### 2. 修改 `syncDocumentState()` - 使用服务器内容作为baseContent

**修复前**：
- 使用本地 `currentContent` 作为 `baseContent`
- 即使检测到服务器版本更新，仍使用本地旧内容

**修复后**：
- **同步前必须获取服务器最新版本**
- **如果服务器版本更新，使用服务器内容作为baseContent**
- 确保不会丢失其他用户的更改

```typescript
// 关键修复：如果服务器版本更新，使用服务器内容作为baseContent
if (serverVersion > currentVersion) {
  // 使用服务器内容作为baseContent，确保不会丢失其他用户的更改
  baseContent = serverContent;
  
  // 更新本地版本号和内容
  this.currentVersion.set(documentId, serverVersion);
  this.currentContent.set(documentId, serverContent);
  
  // 通知编辑器更新内容
  callbacks.forEach(callback => callback(serverContent, serverVersion));
}
```

## 修复效果

### 修复前
1. 用户A打开文档 → 从缓存加载旧数据（只有第一段）
2. 用户A编辑 → 基于旧数据编辑
3. 用户A同步 → 使用旧baseContent，覆盖服务器上的新数据（第一段+第二段）
4. **结果**：第二段丢失 ❌

### 修复后
1. 用户A打开文档 → **从服务器获取最新数据**（第一段+第二段）
2. 用户A编辑 → 基于最新数据编辑
3. 用户A同步 → **使用服务器内容作为baseContent**，正确合并
4. **结果**：所有更改都保留 ✅

## 关键改进点

### 1. 优先服务器策略
- `getDocument()` 总是先尝试从服务器获取
- 只有服务器不可用时，才使用本地缓存
- 使用缓存时，标记为"可能过期"

### 2. 版本检查
- 同步前强制检查服务器版本
- 如果服务器版本更新，使用服务器内容
- 确保不会用旧数据覆盖新数据

### 3. 内容合并
- 使用服务器最新内容作为baseContent
- 确保合并基于最新状态
- 通知编辑器更新内容

### 4. 错误处理
- 服务器获取失败时，使用保守策略
- 版本号减1，强制后端检查
- 记录详细日志，便于调试

## 测试场景

### 场景1：用户A关闭后，用户B编辑
1. 用户A编辑第一段，关闭浏览器
2. 用户B编辑第二段，保存到服务器
3. 用户A重新打开浏览器
4. **预期**：用户A看到第一段+第二段，可以继续编辑

### 场景2：用户A离线编辑
1. 用户A离线编辑第一段
2. 用户B在线编辑第二段
3. 用户A恢复网络，同步
4. **预期**：第一段和第二段都保留

### 场景3：并发编辑
1. 用户A和用户B同时编辑不同段落
2. 用户A先同步
3. 用户B后同步
4. **预期**：两段都保留，正确合并

## 注意事项

1. **网络延迟**：服务器获取有3秒超时，避免阻塞太久
2. **离线支持**：服务器不可用时，仍可使用本地缓存
3. **版本冲突**：后端会进行智能合并，确保不丢失数据
4. **用户体验**：检测到服务器更新时，会通知编辑器更新内容
