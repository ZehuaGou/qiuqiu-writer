# 📖 WriterAI Backend Architecture Overview (开发者阅读指南)

欢迎来到 WriterAI 后端代码库！这份简短的指南旨在帮助新加入的贡献者快速理清本项目的核心 AI 架构逻辑。

## 1. 核心调度：基于 DAG 的 Agent 规划 (Dynamic DAG-based Planning)
长篇小说的创作需要极强的上下文连贯性。本项目没有依赖单次 Prompt 输出，而是构建了一个多 Agent 协同的生产线。
- **工作流拆解**：系统通过有向无环图 (DAG) 管理任务的依赖关系（如：世界观构建 -> 角色生成 -> 大纲规划 -> 章节撰写）。
- **状态流转**：通过维护全局的状态字典 (State)，确保前置 Agent 节点生成的数据能够准确传递给下游节点，实现复杂的逻辑链条。

## 2. 底层引擎：多级记忆系统与 RAG (Multi-level Memory Systems)
为了解决大语言模型在数十万字长文本生成中的“遗忘”和“幻觉”问题，本项目实现了双擎 RAG 架构：
- **向量数据库检索 (例如 Qdrant/Milvus)**：负责非结构化长时记忆的存储与召回。通过 Embedding 相似度，精准提取前文的场景描写和历史事件。
- **图数据库知识图谱 (例如 Neo4j)**：负责结构化实体关系的管理。将小说中错综复杂的人物关系、地理位置等转化为 Graph 存储，在生成新章节前作为强背景约束注入上下文。

## 3. 学习与调试建议
如果你是刚接触本项目的开发者，建议的阅读路径如下：
1. 先查看 `src/memos/mem_scheduler/` 目录，理解任务是如何被分发和调度的。
2. 深入 `src/memos/mem_os/` 和 `core_memories/`，观察大模型是如何与底层数据库进行 RAG 交互的。
