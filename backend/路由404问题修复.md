# 路由404问题修复

## 问题根源

访问 `/api/v1/works` 返回 404 的根本原因是：

**路由注册失败**，因为导入路由时触发了 `memos.__init__.py` 的导入，而 `memos.__init__.py` 需要 `pydantic`，但可能没有安装或环境不对。

## 错误链

```
导入 get_works_router()
  ↓
导入 memos.api.routers
  ↓
导入 memos.api.routers.works_router
  ↓
导入 memos 包（触发 __init__.py）
  ↓
导入 memos.configs.mem_cube
  ↓
需要 pydantic → ModuleNotFoundError
  ↓
路由注册失败（被 try-except 捕获）
  ↓
路由未注册 → 404错误
```

## 解决方案

### 方案1：安装缺失的依赖（推荐）⭐

确保所有依赖都已安装：

```bash
cd backend
pip install pydantic fastapi sqlalchemy
# 或使用 poetry
poetry install
```

### 方案2：修改导入方式（已实现）

已修改 `ai_api.py`，使用 `importlib` 直接导入路由模块，避免通过包导入：

```python
import importlib
works_router_module = importlib.import_module('memos.api.routers.works_router')
works_router = works_router_module.router
app.include_router(works_router)
```

**但这个方法仍然会触发 `memos` 包的初始化**，所以还是需要安装依赖。

### 方案3：延迟导入（备选）

如果依赖确实无法安装，可以：
1. 修改 `memos.__init__.py`，使用延迟导入
2. 或者将路由文件移到不依赖 `memos` 包的地方

## 立即修复步骤

### 1. 安装依赖

```bash
cd backend
pip install pydantic fastapi sqlalchemy asyncpg python-jose passlib
```

### 2. 重启服务

重启服务后，查看启动日志，应该能看到：
```
📋 Works router prefix: /api/v1/works
📋 Works router routes count: X
✅ WriterAI application routers registered successfully
```

### 3. 验证路由

访问 `http://localhost:8001/docs`，应该能看到 `/api/v1/works` 相关的端点。

1. **查看启动日志**：确认是否有路由注册的错误信息
2. **检查依赖**：确认所有依赖都已正确安装
3. **检查环境**：确认使用的是正确的Python环境
4. **查看完整错误**：启动日志中应该有完整的异常堆栈
