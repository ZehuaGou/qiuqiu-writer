# 路由注册问题诊断和修复

## 问题现象

访问 `/api/v1/works?page=1&size=20` 返回 404 错误。

## 根本原因

路由注册失败，因为：
1. 导入 `get_works_router()` 时触发 `memos` 包的初始化
2. `memos.__init__.py` 导入 `memos.configs.mem_cube`
3. `memos.configs.mem_cube` 需要 `pydantic`
4. `pydantic` 可能未安装或不在当前Python环境中
5. 导入失败 → 路由注册失败 → 404错误

## 诊断步骤

### 1. 检查启动日志

查看服务启动时的日志，查找：
- `✅ WriterAI application routers registered successfully` - 成功
- `❌ WriterAI application routers not available` - 失败

### 2. 检查依赖

```bash
# 检查pydantic是否安装

# 检查fastapi是否安装

```

### 3. 检查Python环境

```bash
which python3
python3 --version

# 如果使用poetry
```


## 常见问题

### Q: 为什么直接导入也会失败？

A: Python导入模块时会先导入包（`memos`），触发 `__init__.py`，即使你直接导入子模块。

### Q: 能否避免触发 memos.__init__.py？

A: 可以，但需要修改 `memos.__init__.py` 使用延迟导入，或者将路由移到包外。**但最简单的方法是安装依赖**。

### Q: 为什么其他路由可以工作？

A: 可能其他路由的导入路径不同，或者它们不需要触发 `memos.__init__.py` 的导入。

## 总结

**最直接的解决方案**：安装所有依赖，确保服务在正确的Python环境中运行。

```bash
cd backend
poetry install  # 或 pip install -r requirements.txt
./start_ai_api.sh
```
