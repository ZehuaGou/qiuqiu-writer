# 接口迁移说明

## 📋 迁移概述

已将 `backend/app/api/` 和 `backend/app/services/` 下的接口文件迁移到 `backend/src/memos/api/` 目录下，统一管理所有API接口。

## ✅ 已完成的工作

### 1. 路由文件迁移

**源目录**: `backend/app/api/`  
**目标目录**: `backend/src/memos/api/routers/`

迁移的文件：
- `auth.py` → `auth_router.py`
- `chapters.py` → `chapters_router.py`
- `templates.py` → `templates_router.py`
- `works.py` → `works_router.py`

### 2. 服务文件迁移

**源目录**: `backend/app/services/`  
**目标目录**: `backend/src/memos/api/services/`

迁移的文件：
- `auth_service.py`
- `chapter_service.py`
- `sharedb_service.py`
- `template_service.py`
- `user_service.py`
- `work_service.py`

### 3. 导入路径更新

#### 路由文件中的服务导入
- `app.services.*` → `memos.api.services.*`

#### 服务文件之间的导入
- `app.services.user_service` → `memos.api.services.user_service`

#### 路由前缀更新
所有路由前缀已更新为 `/api/v1/*`：
- `/auth` → `/api/v1/auth`
- `/chapters` → `/api/v1/chapters`
- `/templates` → `/api/v1/templates`
- `/works` → `/api/v1/works`

### 4. 配置文件更新

#### `routers/__init__.py`
- 添加了新路由的导出
- 包含：`auth_router`, `chapters_router`, `templates_router`, `works_router`

#### `ai_api.py`
- 注册了所有WriterAI应用路由
- 更新了根路径的端点信息

#### `app/main.py`
- 更新了路由导入路径
- 更新了ShareDB服务导入

## 🔗 API端点

### WriterAI应用接口

所有接口前缀为 `/api/v1`：

- **认证接口** (`/api/v1/auth/*`)
  - `POST /api/v1/auth/register` - 用户注册
  - `POST /api/v1/auth/login` - 用户登录
  - `POST /api/v1/auth/logout` - 用户登出
  - `POST /api/v1/auth/refresh` - 刷新令牌
  - 等等...

- **章节接口** (`/api/v1/chapters/*`)
  - `POST /api/v1/chapters/` - 创建章节
  - `GET /api/v1/chapters/{chapter_id}` - 获取章节
  - `PUT /api/v1/chapters/{chapter_id}` - 更新章节
  - `DELETE /api/v1/chapters/{chapter_id}` - 删除章节
  - `GET /api/v1/chapters/` - 获取章节列表
  - `POST /api/v1/chapters/{chapter_id}/versions` - 创建版本
  - `GET /api/v1/chapters/{chapter_id}/versions` - 获取版本列表
  - `POST /api/v1/chapters/{chapter_id}/collaborate` - WebSocket协作
  - 等等...

- **模板接口** (`/api/v1/templates/*`)
  - `POST /api/v1/templates/` - 创建模板
  - `GET /api/v1/templates/` - 获取模板列表
  - `GET /api/v1/templates/{template_id}` - 获取模板详情
  - `PUT /api/v1/templates/{template_id}` - 更新模板
  - `DELETE /api/v1/templates/{template_id}` - 删除模板
  - 等等...

- **作品接口** (`/api/v1/works/*`)
  - `POST /api/v1/works/` - 创建作品
  - `GET /api/v1/works/` - 获取作品列表
  - `GET /api/v1/works/{work_id}` - 获取作品详情
  - `PUT /api/v1/works/{work_id}` - 更新作品
  - `DELETE /api/v1/works/{work_id}` - 删除作品
  - 等等...

### AI分析接口

- `/ai/health` - 健康检查
- `/ai/analyze-chapter` - 章节分析
- `/ai/default-prompt` - 默认提示词

### 产品接口

- `/product/*` - 各种产品API

## ⚠️ 注意事项

### 1. 导入路径

**服务文件**仍然使用 `app.` 导入以下模块（这些模块仍在 `app/` 目录下）：
- `app.core.*` - 核心配置、数据库、安全等
- `app.models.*` - 数据模型
- `app.schemas.*` - Pydantic模式

这是**正常**的，因为这些模块属于应用核心，不需要迁移。

### 2. Python路径

确保 `backend/` 目录在 Python 路径中，以便：
- `app.*` 模块可以正常导入
- `memos.*` 模块可以正常导入

### 3. 服务启动

#### 使用 `ai_api.py`（推荐）
```bash
python3 -m memos.api.ai_api --port 8001
```

这将启动包含所有路由的统一API服务。

#### 使用 `app/main.py`（原有方式）
```bash
python3 -m app.main
```

这仍然可以工作，但建议使用 `ai_api.py`。
