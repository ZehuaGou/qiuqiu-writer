# 基于文件名的逐章分析功能使用说明

## 功能概述

这个功能允许你：
1. **首次分析**：使用文件名称自动创建新作品
2. **逐章分析**：每次分析一章时，AI会输出该章的大纲和细纲
3. **自动插入**：分析结果（包括大纲、细纲）自动保存到数据库
4. **卷支持**：支持卷的概念，可以指定卷号
5. **续分析**：第二次分析时，根据文件名自动找到对应的作品并继续插入

## API 接口

### POST /ai/analyze-chapter-by-file

**功能**：根据文件名分析单章并插入到作品

**请求参数**：
```json
{
  "file_name": "我的小说.txt",
  "content": "第一章的完整内容...",
  "chapter_number": 1,
  "volume_number": 1,
  "settings": {
    "model": "gpt-4",
    "temperature": 0.7,
    "max_tokens": 4000
  }
}
```

**参数说明**：
- `file_name` (必需): 文件名，用于查找或创建作品
- `content` (必需): 章节内容
- `chapter_number` (必需): 章节号，从1开始
- `volume_number` (可选): 卷号，默认为1，0表示未分卷
- `prompt` (可选): 自定义分析提示词
- `settings` (可选): AI设置

**响应格式**（流式响应 SSE）：
```
data: {"type": "start", "message": "开始分析第 1 章"}
data: {"type": "work_created", "work_id": 123, "work_title": "我的小说", "message": "创建新作品: 我的小说"}
data: {"type": "chapter_inserted", "chapter_id": 456, "chapter_number": 1, "message": "章节 1 创建成功"}
data: {"type": "done", "message": "分析完成", "data": {...}}
```

**响应数据说明**：
- `work_created`: 创建了新作品
- `work_found`: 找到已存在的作品
- `chapter_inserted`: 章节创建成功
- `chapter_skipped`: 章节已存在，跳过创建
- `done`: 分析完成，包含完整的结果数据

## 使用示例

### 前端调用示例（TypeScript）

```typescript
// 1. 第一次分析（创建作品）
const analyzeChapterByFile = async (
  file_name: string,
  content: string,
  chapter_number: number,
  volume_number: number = 1
) => {
  const response = await fetch(`${API_BASE_URL}/ai/analyze-chapter-by-file`, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${token}`
    },
    body: JSON.stringify({
      file_name,
      content,
      chapter_number,
      volume_number,
      settings: {
        model: 'gpt-4',
        temperature: 0.7,
        max_tokens: 4000
      }
    })
  });

  const reader = response.body?.getReader();
  const decoder = new TextDecoder();

  while (true) {
    const { done, value } = await reader!.read();
    if (done) break;

    const chunk = decoder.decode(value);
    const lines = chunk.split('\n');

    for (const line of lines) {
      if (line.startsWith('data: ')) {
        const data = JSON.parse(line.slice(6));
        
        switch (data.type) {
          case 'start':
            
            break;
          case 'work_created':
            
            break;
          case 'work_found':
            
            break;
          case 'chapter_inserted':
            
            break;
          case 'done':
            
            // data.data 包含完整的结果：
            // - work_id, work_title
            // - chapter_id, chapter_number, volume_number
            // - title
            // - outline (大纲)
            // - detailed_outline (细纲)
            break;
        }
      }
    }
  }
};

// 使用示例
// 第一次分析
await analyzeChapterByFile(
  "我的小说.txt",
  "第一章内容...",
  1,
  1
);

// 第二次分析（继续插入）
await analyzeChapterByFile(
  "我的小说.txt",  // 相同文件名，会找到已存在的作品
  "第二章内容...",
  2,
  1
);
```

## 工作流程

### 第一次分析

1. 用户调用API，传入文件名和第一章内容
2. 后端根据文件名查找作品（从 `work_metadata.source_file`）
3. 如果不存在，创建新作品：
   - 标题 = 文件名（去掉扩展名）
   - 在 `work_metadata` 中存储 `source_file` 和 `analysis_mode: "file_based"`
4. 调用AI分析章节内容（使用 `chapter_analysis` prompt模板）
5. AI返回JSON格式的分析结果，包含：
   - 章节基本信息（标题、章节号、概要）
   - 大纲（outline）：包含 core_function、key_points、visual_scenes、atmosphere、hook
   - 细纲（detailed_outline）：包含 sections 数组
6. 创建章节记录，将大纲和细纲保存到 `chapter_metadata`
7. 在ShareDB中创建文档并保存章节内容
8. 返回结果

### 后续分析

1. 用户调用API，传入相同的文件名和新的章节内容
2. 后端根据文件名找到已存在的作品
3. 检查章节是否已存在（按 work_id + chapter_number + volume_number）
4. 如果已存在，跳过创建并返回现有章节信息
5. 如果不存在，执行AI分析并创建新章节
6. 返回结果

## 数据存储

### 作品（Work）

在 `work_metadata` 中存储：
```json
{
  "source_file": "我的小说.txt",
  "analysis_mode": "file_based"
}
```

### 章节（Chapter）

在 `chapter_metadata` 中存储：
```json
{
  "outline": {
    "core_function": "本章核心功能/目的",
    "key_points": ["关键情节点1", "关键情节点2"],
    "visual_scenes": ["画面1", "画面2"],
    "atmosphere": ["氛围1", "氛围2"],
    "hook": "结尾钩子"
  },
  "detailed_outline": {
    "sections": [
      {
        "section_number": 1,
        "title": "小节标题",
        "content": "小节内容概要"
      }
    ]
  }
}
```

## Prompt 模板

系统使用 `chapter_analysis` 类型的prompt模板，该模板要求AI输出JSON格式，包含：
- 章节基本信息
- 大纲（outline）
- 细纲（detailed_outline）

模板已通过 `init_chapter_analysis_prompt.py` 脚本初始化。

## 注意事项

1. **文件名唯一性**：同一用户下，文件名应该唯一。如果同一文件有多个版本，可以考虑添加版本号或时间戳。

2. **章节号管理**：确保章节号连续且不重复。系统会检查章节是否已存在，避免重复创建。

3. **权限控制**：只能操作自己创建的作品。

4. **错误处理**：如果AI返回的JSON格式不正确，会返回错误信息。

5. **卷支持**：`volume_number` 默认为1，设置为0表示未分卷。

6. **去重逻辑**：系统会自动检查章节是否已存在（按 work_id + chapter_number + volume_number），如果存在则跳过创建。

## 优势

1. **自动化**：无需手动创建作品，根据文件名自动处理
2. **连续性**：支持分多次分析同一文件的不同章节
3. **完整性**：每次分析都包含大纲和细纲
4. **灵活性**：支持卷的概念
5. **去重**：自动避免重复分析同一章节
