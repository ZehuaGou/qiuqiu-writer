# 快速修复路由404问题

## 问题原因

`/api/v1/works` 返回 404 是因为路由注册失败。导入路由时触发了 `memos.__init__.py`，它需要 `pydantic`，但可能没有安装。

## 快速修复

### 方法1：安装依赖（推荐）

```bash
cd backend

# 如果使用 poetry
poetry install

# 或者直接安装
pip install pydantic fastapi sqlalchemy asyncpg python-jose passlib

# 然后重启服务
```

### 方法2：检查启动日志

查看服务启动时的日志，应该能看到：

**如果路由注册成功**：
```
📋 Works router prefix: /api/v1/works
📋 Works router routes count: X
✅ WriterAI application routers registered successfully
```

**如果路由注册失败**：
```
❌ WriterAI application routers not available: ModuleNotFoundError: No module named 'pydantic'
```

### 方法3：临时绕过（不推荐）

如果暂时无法安装依赖，可以修改 `memos.__init__.py`，使用延迟导入：

```python
# 修改前
from memos.configs.mem_cube import GeneralMemCubeConfig

# 修改后（延迟导入）
def _lazy_import():
    from memos.configs.mem_cube import GeneralMemCubeConfig
    return GeneralMemCubeConfig

# 使用时再导入
GeneralMemCubeConfig = _lazy_import()
```

**但这不是好的解决方案**，建议安装依赖。

## 验证修复

1. 重启服务
2. 查看启动日志，确认路由注册成功
3. 访问 `http://localhost:8001/docs`，应该能看到 `/api/v1/works` 端点
4. 测试访问 `/api/v1/works?page=1&size=20`（需要提供认证token）

## 如果仍然404

1. **检查日志**：查看完整的启动日志和错误信息
2. **检查环境**：确认使用的是正确的Python环境和虚拟环境
3. **检查依赖**：运行 `pip list | grep pydantic` 确认已安装
4. **检查导入**：尝试手动导入路由模块，看是否有错误
