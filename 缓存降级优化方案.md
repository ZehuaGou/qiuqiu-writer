# 缓存降级优化方案

## 问题分析

### 当前问题
当数据库查询失败（网络错误、404、服务器错误等）时，系统无法使用本地缓存数据，导致：
- 用户无法查看已缓存的作品数据
- 离线状态下无法工作
- 用户体验差

### 问题根源
1. **`worksApi.getWork()`** 方法在查询失败时直接抛出错误，没有尝试从缓存加载
2. **调用方**（如 `NovelEditorPage`、`WorkInfoManager`）没有缓存降级逻辑
3. **错误处理不完善**，没有区分"数据不存在"和"网络错误"

---

## 优化方案

### 方案一：在 API 层添加缓存降级（推荐）

在 `worksApi.getWork()` 方法中添加缓存降级逻辑：

```typescript
async getWork(
  workId: number,
  include_collaborators?: boolean,
  include_chapters?: boolean,
  check_recovery?: boolean
): Promise<Work> {
  try {
    // 先尝试从数据库获取
    const response = await this.get<any>(`/api/v1/works/${workId}`, {
      include_collaborators,
      include_chapters,
      check_recovery,
    });
    
    const work: Work = {
      ...response,
      work_type: mapWorkTypeToFrontend(response.work_type as BackendWorkType),
    };
    
    // 缓存作品信息
    if (work && work.id) {
      await this.cacheWork(work);
    }
    
    return work;
  } catch (error) {
    // 数据库查询失败，尝试从缓存加载
    
    
    const cacheKey = `work_${workId}_info`;
    const cachedWork = await localCacheManager.get<Work>(cacheKey);
    
    if (cachedWork) {
      
      return cachedWork;
    }
    
    // 缓存也没有，抛出错误
    throw new Error(`无法加载作品 ${workId}：数据库查询失败且缓存中无数据`);
  }
}

private async cacheWork(work: Work): Promise<void> {
  if (work && work.id) {
    const cacheKey = `work_${work.id}_info`;
    try {
      await localCacheManager.set(cacheKey, {
        ...work,
        cached_at: new Date().toISOString(),
      }, 1);
      
    } catch (error) {
      
    }
  }
}
```

### 方案二：在调用方添加缓存降级

在 `NovelEditorPage` 和 `WorkInfoManager` 中添加缓存降级逻辑：

```typescript
const loadWork = async () => {
  try {
    setLoading(true);
    const workData = await worksApi.getWork(Number(workId), true, true);
    setWork(workData);
    setError(null);
  } catch (err) {
    
    
    // 尝试从缓存加载
    const cacheKey = `work_${workId}_info`;
    const cachedWork = await localCacheManager.get<Work>(cacheKey);
    
    if (cachedWork) {
      
      setWork(cachedWork);
      setError('使用缓存数据（数据库不可用）');
    } else {
      
      setError(err instanceof Error ? err.message : '加载作品失败');
    }
  } finally {
    setLoading(false);
  }
};
```

## 实施步骤

### 第一步：优化 `worksApi.getWork()` 方法

1. 添加 try-catch 错误处理
2. 在 catch 中尝试从缓存加载
3. 如果缓存也没有，再抛出错误

### 第二步：优化调用方

1. 在 `NovelEditorPage` 中添加缓存降级
2. 在 `WorkInfoManager` 中添加缓存降级
3. 显示适当的提示信息

### 第三步：测试验证

1. 测试网络断开时的行为
2. 测试数据库返回 404 时的行为
3. 测试缓存数据是否被正确使用

---

## 预期效果

### 优化前
- ❌ 数据库查询失败 → 直接显示错误
- ❌ 无法使用缓存数据
- ❌ 离线状态下无法工作

### 优化后
- ✅ 数据库查询失败 → 自动从缓存加载
- ✅ 缓存数据可用时正常显示
- ✅ 离线状态下可以查看缓存数据
- ✅ 显示适当的提示信息

---

## 注意事项

1. **数据一致性**：缓存数据可能不是最新的，需要提示用户
2. **错误处理**：区分"数据不存在"和"网络错误"
3. **用户体验**：显示清晰的提示信息，告知用户正在使用缓存数据
4. **缓存更新**：确保缓存数据及时更新
