# 前后端对接问题排查指南

## 🔍 问题：前端显示 "错误: Not Found"

### 可能的原因和解决方案

#### 1. 后端服务未启动

**检查方法：**
```bash
# 检查后端服务是否运行
curl http://localhost:8001/docs
# 或者
curl http://localhost:8001/api/v1/works/
```

**解决方案：**
```bash
# 启动后端服务
cd backend
python3 -m memos.api.product_api --port 8001

# 或者使用 ai_api（包含所有接口）
python3 -m memos.api.ai_api --port 8001
```

#### 2. 路由注册失败

**检查方法：**
查看后端启动日志，应该看到：
```
✅ Chapters router registered successfully
✅ Works router registered successfully
```

如果看到警告信息，说明路由注册失败。

**解决方案：**
- 检查 `backend/src/memos/api/routers/works_router.py` 的导入路径是否正确
- 确保所有依赖都已安装
- 检查数据库连接是否正常

#### 3. 认证问题

**检查方法：**
- 打开浏览器开发者工具（F12）
- 查看 Network 标签页
- 检查请求的响应状态码：
  - `401 Unauthorized` - 需要登录
  - `403 Forbidden` - 没有权限
  - `404 Not Found` - 路由不存在

**解决方案：**
- 如果返回 401，需要先登录
- 检查 `localStorage` 中是否有 `access_token`
- 如果 token 过期，需要刷新或重新登录

#### 4. CORS 问题

**检查方法：**
在浏览器控制台查看是否有 CORS 相关错误。

**解决方案：**
- 确保后端已配置 CORS（`product_api.py` 和 `ai_api.py` 都已配置）
- 检查前端请求的 Origin 是否被允许

#### 5. API 路径不匹配

**检查方法：**
- 前端请求：`GET /api/v1/works/`
- 后端路由：`/api/v1/works/`（在 `works_router.py` 中定义）

**已修复的问题：**
- ✅ 修复了 `works_router.py` 中的导入路径（从 `app.core` 改为 `memos.api.core`）
- ✅ 修复了 schema 导入（从 `app.schemas` 改为 `memos.api.schemas`）
- ✅ 修复了前端 URL 构建问题（移除了多余的斜杠）

## 🛠️ 调试步骤

### 步骤 1：检查后端服务

```bash
# 1. 检查后端是否运行
ps aux | grep "product_api\|ai_api"

# 2. 测试后端接口
curl -X GET "http://localhost:8001/api/v1/works/" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json"
```

```bash
# 1. 检查环境变量
cd frontend
cat .env
# 应该包含：VITE_API_URL=http://localhost:8001

# 2. 检查浏览器控制台
# 打开开发者工具，查看 Network 和 Console 标签页
```

### 步骤 3：测试认证

```bash
# 1. 测试登录接口
curl -X POST "http://localhost:8001/api/v1/auth/login" \
  -H "Content-Type: application/json" \
  -d '{
    "username_or_email": "test",
    "password": "test"
  }'

# 2. 使用返回的 token 测试作品接口
curl -X GET "http://localhost:8001/api/v1/works/" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### 步骤 4：检查数据库

```bash
# 检查数据库连接
cd backend
python3 -c "from memos.api.core.database import get_async_db; print('Database OK')"
```

## 📝 常见错误信息

### "Not Found" (404)
- **原因**：路由不存在或未注册
- **解决**：检查后端服务是否启动，路由是否正确注册

### "Unauthorized" (401)
- **原因**：未登录或 token 过期
- **解决**：先登录获取 token

### "Forbidden" (403)
- **原因**：没有访问权限
- **解决**：检查用户权限设置

### "Internal Server Error" (500)
- **原因**：服务器内部错误
- **解决**：查看后端日志，检查数据库连接和依赖

## 📞 如果问题仍然存在

1. 查看后端完整日志
2. 查看浏览器 Network 标签页的请求详情
3. 检查后端和前端代码是否有语法错误
4. 确认所有依赖都已正确安装
