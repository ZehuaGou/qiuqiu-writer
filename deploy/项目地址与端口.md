## 端口占用说明

- **5173**：前端开发（Vite）
- **8001**：后端 API（uvicorn）
- **8080**：Nginx 对外（访问入口，代理到前端静态 + 后端）
- **8000**：若你配置里有 listen 8000，也会被 nginx 使用

当前 `bind() to 0.0.0.0:8080 failed (48: Address already in use)` 表示 **8080 已被占用**（可能是之前已启动的 nginx、或其它程序）。需要先停掉占用 8080（及 8000）的进程，再启动 nginx。

### 查占用并停掉

```bash
# 查看谁占 8080、8000
lsof -i :8080
lsof -i :8000

# 若为 nginx，先停 nginx
nginx -s stop

# 或按 PID 结束（把 PID 换成上面看到的数字）
kill <PID>
```

然后再执行 `nginx`。
